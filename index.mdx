Well-Architected Skills & Steering brings the [AWS Well-Architected Framework](https://docs.aws.amazon.com/wellarchitected/latest/framework/welcome.html) directly into your AI coding agent. Instead of treating architecture reviews as a separate gate, developers get continuous, evidence-backed guidance across all 6 pillars — right in the IDE, at the moment code is written. One set of skills works across 13 supported tools with no AWS credentials required.

## Get started in seconds

<Steps>
  <Step title="Install via npx (auto-detects your tool)">
    ```bash
    npx skills add aws-samples/sample-well-architected-skills-and-steering
    ```
    Auto-detects the AI coding tools configured in your project and installs the right files for each.
  </Step>
  <Step title="Or use a bootstrap script">
    <CodeGroup>
    ```bash macOS / Linux
    curl -sL https://raw.githubusercontent.com/aws-samples/sample-well-architected-skills-and-steering/main/bootstrap.sh | bash
    ```
    ```powershell Windows
    & ([scriptblock]::Create((irm https://raw.githubusercontent.com/aws-samples/sample-well-architected-skills-and-steering/main/bootstrap.ps1)))
    ```
    </CodeGroup>
  </Step>
  <Step title="Ask your AI agent a Well-Architected question">
    ```text
    What Well-Architected pillars should I consider for this architecture?
    ```
    If configured correctly, your agent will reference all six pillars with specific guidance rather than a generic answer.
  </Step>
  <Step title="Run a full review (optional)">
    For Claude Code users, invoke the slash command directly:
    ```text
    /wa-review
    ```
    For Kiro users, steering loads automatically — just mention "architecture review" and WA principles apply.
  </Step>
</Steps>
