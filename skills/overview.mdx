Skills are self-contained, tool-agnostic playbooks that any AI agent can follow step-by-step. They don't depend on steering files or on each other — each skill is a complete, standalone guide that embeds AWS Well-Architected Framework best practices directly into your development workflow. Drop one into any supported AI coding tool and invoke it with a natural-language prompt.

The table below maps each skill to the Well-Architected pillar(s) it covers and the scenario it's designed for.

## How to Invoke Skills

Skills are invoked with natural-language prompts. Each tool exposes them differently — some use slash commands, others load them as rules or instructions that the agent applies automatically.

<Tabs>
  <Tab title="Claude Code">
    Claude Code exposes skills as slash commands. After installing the adapter, invoke any skill directly from the chat input:

    ```text
    /wa-review
    /security-assessment
    /reliability-improvement-plan
    /cost-optimization-review
    /performance-efficiency
    ```

    You can also use natural-language prompts in any conversation:

    ```text
    Run a WA review on this codebase
    Do a security review — focus on IAM and encryption
    Find single points of failure in our infrastructure
    Check our cost optimization posture
    ```
  </Tab>
  <Tab title="Kiro">
    Kiro loads skills automatically from `.kiro/skills/`. Start any conversation using the skill's trigger phrases and Kiro applies the playbook without an explicit command:

    ```text
    Do a well-architected review of this project
    Assess our security posture
    Find reliability gaps in the infrastructure
    Review our cost configuration
    ```

    The **Kiro Power** (recommended) bundles skills with steering and keyword-based auto-activation — mention "architecture review", "security review", or "reliability" and Kiro loads the right skill automatically.
  </Tab>
  <Tab title="Cursor / Windsurf / Cline">
    Skills are loaded as rules and applied contextually. Use natural-language prompts in the chat panel:

    ```text
    WA review this project
    Security review — check IAM policies and encryption
    What reliability risks do you see?
    Identify cost optimization opportunities
    ```
  </Tab>
  <Tab title="Codex / Gemini CLI / Amp / OpenClaw">
    Skills are referenced from the `skills/` directory. Invoke them with natural-language prompts:

    ```text
    Run a well-architected review
    Assess security posture for this workload
    Reliability improvement plan for this service
    Cost optimization review
    ```
  </Tab>
  <Tab title="GitHub Copilot">
    Skills are embedded in `.github/copilot-instructions.md`. Use natural-language prompts in Copilot Chat:

    ```text
    @workspace Do a WA review
    @workspace Security assessment — focus on IAM and network security
    @workspace Find cost optimization opportunities
    ```
  </Tab>
  <Tab title="AWS DevOps Agent">
    Skills are packaged as `.zip` files and uploaded to your Agent Space via the Operator Web App. Once uploaded, invoke them naturally:

    ```text
    Run a well-architected review on this workload
    Security assessment for the current deployment
    ```
  </Tab>
</Tabs>

## Skill vs Steering

<Note>
  Skills and steering serve different purposes. **Skills** are explicit playbooks — you invoke them for a specific task and the agent follows a structured, multi-step process. **Steering** is always-on context that shapes how the agent reasons about architecture in every conversation, without an explicit invocation. Most tools use both together for the best results.
</Note>

| | Skills | Steering |
|---|---|---|
| **When active** | When explicitly invoked | Every conversation |
| **Scope** | One focused task | Continuous background guidance |
| **Depth** | Deep, structured, step-by-step | Lightweight, principle-level |
| **Output** | Formal report or plan | Inline suggestions and corrections |
| **Tools** | All 13 supported tools | Kiro (native), others via adapters |
